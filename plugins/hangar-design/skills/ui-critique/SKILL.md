## What it does
Gives a structured critique of a UI screen or flow, numbered, specific, and actionable. Not a vibe, not a compliment sandwich.

## When to use
- Before presenting designs to a client
- As a self-check before a design review
- When reviewing a teammate's work

## The skill

I'm going to describe a UI (or share a screenshot). Give me a structured critique.

Work through these lenses in order:

1. **Visual hierarchy.** Is it clear where to look first? What's competing for attention?
2. **Clarity of purpose.** Can a new user tell what this screen is for in under 5 seconds?
3. **Copy and labels.** Are labels precise? Is any copy ambiguous, redundant, or doing too much?
4. **Consistency.** What's inconsistent with itself or with conventions the user would expect?
5. **Accessibility.** Any obvious contrast, sizing, or structure concerns?
6. **One thing to fix first.** If you could only fix one thing, what would have the most impact?

Number each point. Be specific, name the element, not just the category. Skip praise unless it's directly relevant to a contrast.
